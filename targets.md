Plan
既然你已经理解了底层的逻辑，那我们现在就把这个“仿小红书”项目拆解成一个为期 4 周的寒假突击计划。
这个计划表是按照先跑通、后优化、再加亮点的面试思路设计的。

📅 第一周：地基搭建（用户系统与环境）
目标： 完成开发环境配置，实现最基础的注册登录。
* Day 1-2：环境与规范
    * 安装 Go, MySQL, Redis, Postman。
    * 学习 Go 工程目录结构（建议采用 cmd/, internal/, api/ 这种标准结构）。
    * 练习： 使用 GORM 连接 MySQL，自动生成用户表（User Table）。
* Day 3-4：用户接口
    * 实现注册接口：密码存储必须使用 bcrypt 哈希加密，绝对不能存明文。
    * 实现登录接口：学习 JWT (JSON Web Token) 的签发和验证。
* Day 5-7：中间件与鉴权
    * 编写一个 AuthMiddleware：所有需要登录的接口都要经过它。
    * 产出： 你可以用 Postman 注册账号、登录并拿到 Token。

📅 第二周：核心业务（笔记与内容）
目标： 实现小红书最核心的“发笔记”和“看笔记”。
* Day 1-2：笔记模型设计
    * 设计 Note 表：支持标题、正文、图片链接。
    * 练习： 完成 CreateNote 和 GetNoteDetail 接口。
* Day 3-4：图片上传（接入 OSS）
    * 重点： 学习如何对接阿里云 OSS 或 MinIO。
    * 流程：前端传图片 -> 后端接收 -> 转存 OSS -> 数据库存 URL。
* Day 5-7：点赞与收藏（引入 Redis）
    * 练习： 用户点赞一条笔记，数据先存入 Redis 的 Set 中。
    * 思考：如何防止用户对同一条笔记重复点赞？

📅 第三周：社交属性（关系与信息流）
目标： 让你的 App 动起来，实现“关注”和“动态”。
* Day 1-2：关注系统
    * 实现“关注/取关”接口。
    * 难点： 查询“我的关注列表”和“我的粉丝列表”。
* Day 3-5：Feed 流（关注动态）
    * 练习： 实现一个简单的接口，拉取我关注的所有人发布的笔记，并按时间倒序排列。
* Day 6-7：评论系统
    * 实现一级评论和二级回复。
    * 思考： 数据库表如何设计才能支持“楼中楼”查询？

📅 第四周：面试加分项（优化与总结）
目标： 解决高性能问题，完善简历描述。
* Day 1-2：性能优化
    * 缓存同步： 编写一个定时任务（Cron Job），把 Redis 里的点赞数同步回 MySQL。
    * 数据库优化： 给 user_id、note_id 等高频查询字段加索引。
* Day 3-4：搜索增强
    * （可选）如果精力允许，尝试接入 Elasticsearch 实现简单的关键词搜索。
* Day 5-6：文档与测试
    * 编写接口文档（Swagger 或简单的 Markdown）。
    * 编写 1-2 个核心逻辑的单元测试（Unit Test）。
* Day 7：简历打磨
    * 将项目写在简历上，重点突出：JWT 鉴权、OSS 存储、Redis 优化点赞性能、Feed 流设计。

💡 给你的几个实战建议：
1. 不要卡在 UI 上： 你是后端开发，不需要做精美的 App 界面。用 Postman 或 Apifox 调试通接口即可。
2. 小步快跑： 先保证 go run main.go 能跑起来，再考虑代码优不优雅。
3. 多看文档： 学习 GORM 官方文档 和 Gin 官方文档，它们是你寒假最好的老师。
